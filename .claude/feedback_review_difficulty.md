---
name: Review questions must be challenging
description: Review-based learning questions should be system-level and difficult, not surface-level code mechanics
type: feedback
---

Review questions need to be hard enough to be useful. At Stage 5+, Noor already knows Vue composables, routing, CRUD, SQL queries, auth middleware. Don't ask about syntax or "what does this do" — ask about design decisions, cross-component consequences, and system-wide data flow.

**Why:** Easy questions don't surface real gaps. The whole point of review-based learning is to find what Noor *doesn't* yet understand — not to confirm what's obvious.

**How to apply:** Every Why/What-if/Trace question should require reasoning about tradeoffs, failure modes, or multi-layer interactions. If the answer is "just read the code," the question is too easy.
