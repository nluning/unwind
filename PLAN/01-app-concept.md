# Unwind — App Concept

## The problem

People with neurodivergent brains (gifted, autistic, ADHD, or combinations) often
struggle to "switch off." When the workday ends or stress builds up, it's hard to
choose a relaxing activity — the paradox of choice kicks in, or the brain keeps
spinning in the same gear it was already in.

Common patterns:

- Knowing you need to relax but not knowing _what_ to do
- Picking an activity that matches your current mental mode instead of
  counterbalancing it (e.g., doing more screen work to "relax" after a screen-heavy day)
- Open-ended free time feeling overwhelming rather than restful
- Stress making it harder to think clearly about what would reduce that stress

## The idea

Unwind is a simple, minimal app that suggests activities to help the user relax.
It removes the decision burden by narrowing choices down to one suggestion at a time.

The app maintains a personalized activity database per user. Activities come from
three sources:

- A shared base list that every user starts with
- Activities the user adds manually
- Activities generated by AI, tailored to the user's preferences

Every activity is tagged with metadata that makes filtering possible:

- **Categories**: Head (cognitive), Hands (physical/creative), Heart (social/emotional)
  — an activity can belong to multiple categories
- **Stress range**: suitable for stress levels 1-5
- **Suggested duration**: a timeframe in minutes, because open-endedness often
  doesn't work for the target audience

## How the app works

When the user opens the app, it **defaults to Mode 1 — "Just suggest something."**
One tap, one suggestion. No decisions required. This is deliberate: the target user
may be at a 9/10 stress level and unable to process a menu of options.

The other modes are accessible but not upfront — tucked behind a simple navigation
element. The user discovers them when they're ready, not when they're overwhelmed.

### Mode 1 — "Just suggest something" (default)

The simplest mode. User taps a button, gets a random activity from their personal
database. Don't like it? Tap again. Keep going until something clicks.

No AI involved — pure randomization from the user's list.

### Mode 2 — "I'm stressed"

User selects their stress level (1-5). The app suggests a random activity that is
suitable for that stress level. Higher stress = activities that are simpler, shorter,
and more grounding.

No AI involved — database filtering + randomization.

### Mode 3 — "Counterbalance my day"

User indicates what kind of activity they've been doing (Head / Hands / Heart).
The app suggests an activity from a _different_ category to counterbalance.

For example: after a Head-heavy day of desk work, the app suggests a Hands activity
(go for a walk, do some drawing) or a Heart activity (call a friend, play with
a pet).

No AI involved — category filtering + randomization.

### Mode 4 — "Help me figure it out"

A short conversation with an AI that helps the user find the right activity through
simple yes/no questions and limited choices. Designed to be low-effort — the user
shouldn't have to think hard or type long answers.

Activities suggested in this mode can be added to the user's personal database,
so the AI effectively helps grow and refine the list over time.

## Personalization over time

The app tracks which suggestions the user accepts and which they skip. Over time,
this data informs better suggestions — surfacing activities the user tends to
accept and de-prioritizing ones they consistently skip.

**The cold start problem:** personalization based on accept/skip tracking takes
many interactions to produce meaningful signal. A user who opens the app twice a
week won't notice smarter suggestions for weeks or months. To mitigate this:

- The AI onboarding conversation generates a genuinely personalized starter list
  (not just the generic base list)
- Users can explicitly remove activities that don't suit them ("never suggest this
  again") — this is faster signal than tracking skips
- The base list is curated to be broadly useful, so even without personalization
  the suggestions have value

Optionally, after completing an activity, the user can rate how they feel compared
to before (a simple before/after stress comparison). This feedback loop helps
validate which activities actually work for stress reduction, not just which ones
sound appealing in the moment.

## Design principles

- **Minimal**: no visual clutter. The user is already overwhelmed — the app should
  feel calm.
- **One thing at a time**: never show a list of options. One suggestion, accept or
  skip. The mode selection itself follows this principle — Mode 1 is the default,
  other modes are discoverable but not presented as a choice upfront.
- **Timeboxed by default**: every activity comes with a suggested duration.
  "Go for a walk" becomes "Go for a 20-minute walk."
- **Three visual themes**: calm, warm, playful — user's choice. Small touch that
  makes the app feel personal.
- **Offline-capable**: modes 1-3 should work without internet. Mode 4 requires
  a connection.

## Why not use an existing app?

Several apps address adjacent problems: Calm and Headspace for meditation/wellness,
Finch for gentle habit building, Tiimo for ADHD-friendly scheduling, Forest for
focus. None of them do exactly what Unwind does:

- **Meditation apps** assume relaxation = mindfulness. For many neurodivergent
  people, sitting still is the opposite of relaxing.
- **Habit trackers** are about building routines, not answering "what do I do
  right now?"
- **Scheduling apps** require planning ahead, which is exactly what the target
  user struggles with in the moment.

Unwind is specifically about the in-the-moment question: "I need to switch off
and I can't figure out how." It's also a learning project — building it teaches
full-stack development, AI integration, and deployment skills that are valuable
regardless of whether the app finds a large audience.

## Future possibilities

- Integration with an energy-tracking app (being developed separately by a
  colleague) to factor in energy levels alongside stress
- Multi-language support (built i18n-ready from the start, Dutch only initially)
- More sophisticated AI personalization based on accumulated usage patterns
- Optional timer so as to not get lost in an activity
- More categories next to Head/Hands/Heart, such as: chores, repetitive activities, low-stimuli/high-stimuli, et cetera

## What this app is NOT

- It is not a therapy tool or medical device
- It does not diagnose or treat any condition
- It does not require or store clinical health data
- It is a practical, lightweight tool for a common everyday problem
